Attendance Management System

An enterprise-grade platform for employee attendance, leave, task management, and departmental administration — built on Laravel 11.

Version 1.0 Framework Laravel 11 Date 2026-03-17 Roles Admin · Manager · Employee
📋

Overview

The Attendance Management System (AMS) is a centralized web application that helps organizations manage employee attendance, leaves, permissions, tasks, and departmental configurations. It supports three role-based user types with tailored dashboards and strict access controls.

📍
GPS Punch Tracking Real-time attendance with location data
🌿
Leave Workflows Multi-level approval chains for all leave types
✅
Task Management Assign, track, and collaborate on tasks
🔔
Notifications In-app alerts for all key events
🕐
Auto Leave Credits Scheduled cron jobs for leave management
🔒
ID Encryption AES-128 encrypted route parameters
👥

User Roles

The system enforces role-based access control via middleware, with automatic dashboard redirection after login.

🛡️ Admin

Full system access. Manages employees, departments, holidays, leave rules, site settings, and cron jobs.

👔 Manager

Manages team attendance, approves leave requests, assigns tasks, and monitors team performance.

🧑‍💻 Employee

Views personal attendance, submits leave/permission requests, and updates assigned task statuses.

🔐

Authentication

ℹ️ Users can log in using their email address, username, or employee code — whichever they prefer.
  • Multi-identifier login: email, username, or employee code
  • Remember-me functionality for persistent sessions
  • Automatic role-based dashboard redirection after login
  • Secure session regeneration on login and logout
  • Strong password requirements: uppercase, lowercase, digit, special character
  • New employees receive their initial password via email
🧑‍💼

Employee Management

🛡️ Admin only
  • Full CRUD — Create, view, edit, and manage all employee records
  • Auto Employee Code Generation — Configurable prefix (e.g., EMP0001)
  • Two-Level Manager Hierarchy — Each employee has an L1 and L2 manager
  • Role Assignment — Set as employee or manager
  • Employment Status — Toggle between active and inactive
  • Profile Management — Personal details and profile photo upload

Leave & Permission System

👥 All roles — submit  |  👔 Manager & 🛡️ Admin — approve

Leave Types

TypeDescriptionKey Rules
Casual Leave (CL) Date range leave request Annual balance tracked; monthly cap (default 3/month); auto-fallback to LOP if balance is zero
Permission Short leave in hours for a single date Monthly limit per department; configurable hour duration; no duplicate dates
Saturday Leave For flexible-Saturday departments One-time per month; auto-expires if unused by month end

Two-Level Approval Workflow

Employee Submits
→
L1 Approves
→
L2 Final Approval
→
Approved
  • Manager remarks recorded at each approval step
  • Auto-advances to final if no L2 manager is assigned
  • Admin handles approvals if no managers are configured
  • Rejection possible at any stage

Leave Balance & Audit Trail

  • LeaveBalance — tracks credited, used, and lapsed amounts
  • LeaveTransaction — complete audit trail: credit, debit, lapse
  • Attachment support (max 4 MB): jpg, jpeg, png, pdf, doc, docx

Cron Jobs & Scheduling

🛡️ Admin only

Scheduled Commands

leave:credit-monthly  Monthly

  • Credits CL based on department configuration
  • Credits monthly permission allowance
  • Credits Saturday leave if eligible
  • Expires previous month's unused permissions and Saturday leave
  • Duplicate-safe (won't re-credit the same period)
  • Supports --dry-run flag for verification

leave:year-end-reset  Dec 31

  • Lapses all remaining CL balances
  • Creates audit entries in LeaveTransaction
  • Supports --dry-run flag

Cron Job Management UI

  • List all scheduled jobs with last run time and status
  • Manual job execution from the admin interface
  • Execution logs with output capture, duration (ms), and error details
  • Color-coded status indicators

Architecture Notes

  • Trait HasEncryptedRouteKey — applied to all models to encrypt primary keys in URLs
  • Helper IdCrypt — URL-safe AES encryption/decryption utility for integer IDs
  • Middleware RoleMiddleware — enforces role-based access on protected route groups
  • AJAX Interactions — calendar data loading, notification reads, and punch state checks
  • Email — initial password delivery and event notifications via Laravel's mail system
